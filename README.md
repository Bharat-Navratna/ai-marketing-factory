# AI Campaign Studio

Production-ready full stack AI marketing application that turns a campaign brief into structured, export-ready Meta and TikTok campaign assets.

## Live Demo

The application is deployed on AWS EC2:

http://18.168.199.197

## Project Overview

AI Campaign Studio uses a custom TypeScript orchestration pipeline to coordinate specialized AI agents for market research, strategy, copywriting, channel planning, and quality review. The frontend provides a polished Next.js dashboard for entering a brief, tracking generation progress, reviewing campaign outputs, and exporting ad packages as JSON.

The production deployment runs two Node.js processes behind Nginx on Ubuntu Server 24.04 LTS:

- Next.js frontend on port `3002`
- Express TypeScript API on port `3001`
- Nginx reverse proxy on port `80`
- PM2 process manager for uptime and restarts

## Features

- Full campaign generation from a structured marketing brief
- Five-stage AI workflow: research, strategy, copywriting, channel planning, review
- Gemini API integration through LangChain's OpenAI-compatible client
- Zod schemas for validated LLM inputs and outputs
- Export-ready Meta Ads and TikTok Ads packages
- Campaign dashboard with overview, research, strategy, copy, ads packages, email, calendar, KPIs, and export views
- JSON export and clipboard actions for generated assets
- Local CLI support for legacy campaign generation
- Saved campaign output in local JSON files
- Production deployment on AWS EC2 using PM2 and Nginx

## Architecture Overview

```text
Browser
  |
  | HTTP
  v
Nginx reverse proxy
  |
  | /              -> Next.js frontend, port 3002
  | /api/*         -> Express API, port 3001
  v
Express API
  |
  | POST /api/campaigns/generate
  v
Campaign Orchestrator
  |
  | 1. Research Agent
  | 2. Strategy Agent
  | 3. Copy Agent
  | 4. Channel Planning Agent
  | 5. Review Agent
  v
FullCampaign JSON + export package
```

## API Workflow

1. The user submits a campaign brief from the Next.js frontend.
2. The frontend calls `POST /api/campaigns/generate`.
3. The Express API validates the request with `CampaignInputSchema`.
4. The orchestrator runs the five AI agents in sequence.
5. Each agent returns schema-validated structured output.
6. The API saves the full campaign to `output/full/`.
7. The frontend renders the campaign dashboard and export package.

## Local Development Setup

### Prerequisites

- Node.js 18 or newer
- npm
- Gemini API key from Google AI Studio

### Backend

```bash
npm install
cp .env.example .env
```

Add your Gemini API key to `.env`:

```bash
GEMINI_API_KEY=your-gemini-api-key
```

Start the API:

```bash
npm run api
```

The backend runs at `http://localhost:3001`.

### Frontend

```bash
npm run frontend:install
npm run frontend
```

The frontend runs at `http://localhost:3002`.


## API Reference

### `POST /api/campaigns/generate`

Generates a full campaign using the five-agent orchestrator.

```json
{
  "brandName": "Lumiere",
  "productDescription": "A luxury anti-aging serum for mature skincare buyers.",
  "industry": "Beauty and Skincare",
  "objective": "sales_conversion",
  "targetAudience": "Women aged 35 to 55 interested in premium skincare.",
  "region": "United States",
  "tone": "luxury",
  "platforms": ["meta_ads", "instagram"],
  "competitors": "La Mer, SK-II",
  "budget": "$5,000/month",
  "duration": "4 weeks"
}
```

Valid objectives: `brand_awareness`, `lead_generation`, `sales_conversion`, `app_installs`, `engagement`, `retargeting`

Valid tones: `professional`, `playful`, `luxury`, `urgent`, `inspirational`, `educational`

Valid platforms: `meta_ads`, `instagram`, `facebook`, `tiktok`, `linkedin`

### `GET /api/campaigns`

Returns saved full campaigns from `output/full/`.

### `GET /api/health`

Returns API health status and timestamp.

### `POST /api/generate`

Legacy endpoint for the older product and audience flow.

### `GET /api/campaigns/legacy`

Returns campaigns generated by the legacy pipeline.

## Nginx Reverse Proxy Setup

Example Nginx server block:

```nginx
server {
    listen 80;
    server_name 18.168.199.197;

    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://127.0.0.1:3002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Validate and reload Nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
```
